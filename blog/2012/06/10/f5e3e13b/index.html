Octopress導入への道

Jun 10th, 2012

github pagesへのOctopressの導入をツイッターでつぶやきつつ作業したので、 それをはりつけて振り返ってみる。

発端

記事をgitでcommitすれば勝手に更新されるブログシステムが欲しいです。あと導入が簡単で、自由度高すぎず、おしきせのそこそこなスタイルがいくつかから選べるということなし

— らっしーさん (@lass1e) 6月 9, 2012

Octopressのインストール

octopress.org/docs/setup/ を参照しながら環境をセットアップ開始

— らっしーさん (@lass1e) 6月 9, 2012

git やrvmはすでにはいっていたが、octopressをcloneしたところにcdしたら.rvmrcがあって1.9.2を指定している。1.9.3じゃ駄目なのかな

— らっしーさん (@lass1e) 6月 9, 2012

1.9.3でもいいような気がするので、rvm use 1.9.2を1.9.3に編集してとりあえずcommit

— らっしーさん (@lass1e) 6月 9, 2012

bundleもすでにはいっているので、bundle install –path vendor/bundleでgemをごそっともってくる

— らっしーさん (@lass1e) 6月 9, 2012

rake installでいろいろ設定された

— らっしーさん (@lass1e) 6月 9, 2012

これはdefaultのclassicテーマを使うためにいろんなファイルをコピーする作業だった。後ですこし試してみたが、他のテーマをダウンロードしてきてからrake install[“テーマ名”]でそのテーマで必要とされるファイルがごそっとコピーされてテーマが切り替わる。

github pagesの設定

githubのプロジェクトページの作りかたを調べる

— らっしーさん (@lass1e) 6月 9, 2012

githubのアクセスがなんかみょうなのでgitの最新版をいれなおす

— らっしーさん (@lass1e) 6月 9, 2012

てきとうにファイル一個追加してリポジトリ作ってからの、Admin->githubpageつくる、みたいなボタンをぽちっとおしたらプロジェクトページできた

— らっしーさん (@lass1e) 6月 9, 2012

おしきせのファイルがいっぱいできたが、どうせいらなくなるだろうからごそっとrmしてpushしたら pageつくれねえよ、ってエラーメールがとんできたが気にしない

— らっしーさん (@lass1e) 6月 9, 2012

Octopressとgithub pagesの連携

ここでoctopressにもどる

— らっしーさん (@lass1e) 6月 9, 2012

(bundle exec) rake setup_github_pages を実行するもhttpsなリポジトリ指定では駄目だった sshなほうならOK

— らっしーさん (@lass1e) 6月 9, 2012

というわけでさっき作ったblogリポジトリを指定したら、octopressの下位ディレクトリに、blogリポジトリ用のgitワーキングディレクトリができた これっていいんだっけ submoduleなのか?

— らっしーさん (@lass1e) 6月 9, 2012

上位のリポジトリで下位のリポジトリがまるごと無視設定されてるからいいのかな なるほどこういうやり方もあんのか

— らっしーさん (@lass1e) 6月 9, 2012

rake generate ,rake deployしたら lassie116.github.com/blog/ にからっぽのができたOctopressなサイトができた

— らっしーさん (@lass1e) 6月 9, 2012

octopress自体をmasterにしてる | そこで作業しているのは筋がわるそうなので自分octopress用のbranchを作ってそこで管理しよう

— らっしーさん (@lass1e) 6月 9, 2012

初記事の投稿

環境ととのったような気がしないでもないので初記事をかく

— らっしーさん (@lass1e) 6月 9, 2012

ここでこの記事やこの記事を書く。rake new_postでファイル生成して、markdownで文章書いて、rake generateで記事生成して、rake deployでgithubにpush。Webサイトの編集ページを開く必要もなく、いつものエディタで文書を編集した後にぽちぽちっとコマンドたたくだけで更新できるので非常に便利。

あーまーたしかにここまで出来てしまえば悪くない気はするね。あと記事のURL決めるのめんどいので適当なIDを勝手に生成してほしい

— らっしーさん (@lass1e) 6月 9, 2012

記事を書く前に考えないといけないことがあると書く気力がなくなるので、英文でタイトルを付ける必要をなくしたい。この記事で書いたようにRakefileに仕掛けをしてnew_postで適当なタイトルを付けるようにした。

食わず嫌いはいけません

作業前

octopressは実行環境なんだか必要なリソースなんだか自分の記事なんだかがいっしょくたになってるのが非常に好みではない。

— らっしーさん (@lass1e) 6月 9, 2012

作業後

生成環境も含めてまるごとgitリポジトリにつっこんでおいて、cloneさえできればどこからでも生成できるのがいいところなのか。それでスクリプト含めて全部まるかかえしてるんだな。たいした容量でもないしこれはこれでありかそうか

— らっしーさん (@lass1e) 6月 9, 2012

この間わずか1時間半。

次回予告

octopressは慣れてくると確かにけっこう楽しいな フレームワークが二重にかぶってる上にそれぞれのレイヤーの説明があんまりないのでわかりづらいけど

— らっしーさん (@lass1e) 6月 9, 2012

HTMLファイル群(=Webサイト)生成ツールであるJekyllと、それ使ってテーマ指定やgithub pagesとの連携を管理するOctopress。それぞれの役割を理解せずに、いきなりOctopressを導入しはじめると何がどうなっているのかよくわからずに混乱しやすいので、次回はこの二つのフレームワークの役割を説明する。
